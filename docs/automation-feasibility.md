# 菜鸟云仓库存自动化可行性

更新时间：2026-06-06

## 当前结论

可行但不能完全无人值守，原因是菜鸟登录/滑块风控。

## 已验证

- 钉钉机器人发送可用。
- 库存文件整理可用，低库存阈值为 10。
- 使用本地 Chrome 已登录状态时，可以进入库存多维查询页。
- Codex Chrome Extension 重新握手后可连接 Chrome。

## 不采用

- 不再使用 macOS 鼠标点击方案，因为会影响用户正常使用电脑。
- 不使用 Codex 内置浏览器作为长期方案，因为它没有用户本地 Chrome 的菜鸟登录态，会停在登录页。
- 不用 Playwright 专用浏览器档案绕滑块，菜鸟风控不稳定。

## 推荐执行方式

1. 用户先在本地 Chrome 中保持菜鸟云仓登录。
2. 用户在 Codex 中发“执行”。
3. Codex 通过 Chrome 插件后台接管已登录标签页，导出库存、整理数据并发送钉钉。

## 每日自动化限制

普通 macOS `launchd` 不能调用 Codex 的 Chrome 插件，也不能共享 Codex 会话上下文。因此，在“不抢鼠标、不绕验证码”的约束下，完全无人值守的每日自动执行暂不可稳定实现。
